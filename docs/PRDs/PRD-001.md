## 1. Tổng quan

`prd:databuckets-sdk-skills-0001`

Tạo bộ **agent skills** cho Databuckets Unity SDK v1.0.6, giúp AI agent tự động hiểu và sử dụng đúng các API của SDK khi được yêu cầu. Mỗi API là một skill riêng biệt, tuân theo chuẩn `skill-creator`.

- **Đối tượng sử dụng:** AI Agent
- **SDK version:** v1.0.6
- **Mức độ skill:** Standard (300–800 từ)
- **Chuẩn viết:** Theo `skill-creator/SKILL.md` và `docs/contributors/skill-template.md`

### Cấu trúc mỗi skill (theo skill-creator)

Mỗi skill PHẢI có đầy đủ các sections theo thứ tự:

1. **Frontmatter** — name (kebab-case, khớp tên thư mục), description (dưới 150 ký tự)
2. **Title (H1)** — Tên skill mô tả
3. **Overview** — Mục tiêu + giải thích 2-4 câu (WHY)
4. **Khi nào sử dụng / Khi KHÔNG sử dụng** — Scenarios cụ thể
5. **Điều kiện tiên quyết** — Init phải gọi trước (trừ skill init)
6. **Hướng dẫn thực hiện** — Từng bước, động từ mệnh lệnh (HOW)
7. **Ví dụ** — 1 ví dụ thực tế với code chạy được (WHAT)
8. **Best Practices** — ✅ Nên / ❌ Không nên
9. **Anti-Patterns** — Lỗi phổ biến + cách sửa
10. **Skill liên quan** — Liên kết với skill khác trong bộ

## 2. Tổng quan SDK

`prd:sdk-overview-0002`

Databuckets Unity SDK là nền tảng analytics tự động thu thập event, session, và app lifecycle trong Unity.

### Tự động (Automatic)
- Session tracking (`session_start`)
- App lifecycle tracking (`app_focus_start`, `app_focus_end`)
- User ID và geo info setup
- Device info + App info collection
- First open tracking (`first_open`)

### Thủ công (Manual)
- Business event tracking
- User property updates
- Configuration changes

### Auto-Injected Fields

SDK tự động thêm vào mọi event:

| Nhóm | Trường | Mô tả | Ví dụ |
|------|--------|-------|-------|
| Time | `event_date` | Ngày event (YYYYMMDD) | `20250505` |

| Time | `event_local_hour` | Giờ (0-23) | `11` |

### Auto-tracking Events

| Event | Khi nào gửi | Ghi chú |
|-------|-------------|---------|
| `first_open` | Mở app lần đầu trong 10 phút sau cài | Gửi 1 lần duy nhất |
| `session_start` | Session mới: app launch, restart, timeout, resume | Quản lý bởi SessionUtil |
| `app_focus_start` | App được focus | Tính foreground duration |
| `app_focus_end` | App mất focus, minimize, crash, quit | Tính foreground duration |
| `app_exception_log` | Unity LogType.Exception | Chỉ khi EnableExceptionLogTracking bật |

### PlayerPrefs Keys (KHÔNG xóa)

| Key | Mô tả |
|-----|-------|
| `is_sent_first_open` | Đánh dấu đã gửi first_open |
| `session_number` | Số session hiện tại |
| `user_pseudo_id` | Anonymous user ID |
| `current_session_start` | Thời điểm bắt đầu session |
| `last_event_timestamp` | Timestamp event cuối cùng |

> ⚠ **Cảnh báo:** Xóa các PlayerPrefs keys này mà không backup sẽ làm sai lệch dữ liệu analytics.

## 3. Danh sách Skills (Features)

<a id="feature-skill-init-0001"></a>

### 3.1. Skill: databuckets-init

`feature:skill-init-0001`
> Implements: [`prd:databuckets-sdk-skills-0001`](#prd-databuckets-sdk-skills-0001)

- **API:** `DatabucketsTracker.Init(apiEndpoint, apiKey)`
- **Mô tả:** Khởi tạo SDK. BẮT BUỘC gọi 1 lần duy nhất trước khi dùng bất kỳ API nào khác.
- **Tham số:**
  - `apiEndpoint` (string): URL endpoint của Databuckets API
  - `apiKey` (string): API key xác thực
- **Lưu ý quan trọng:**
  - PHẢI gọi sau khi scene loaded (KHÔNG gọi trong `Awake()`)
  - Cách 1: Dùng Coroutine chờ 1 frame
  - Cách 2: Dùng delay 100ms
- **Ví dụ:**
  ```csharp
  DatabucketsTracker.Init("api-endpoint-here", "your-api-key-here");
  ```
- **Best Practices:**
  - ✅ Gọi trong `Start()` hoặc sau `yield return null`
  - ✅ Chỉ gọi 1 lần duy nhất trong toàn bộ app lifecycle
  - ✅ Gọi trước mọi API khác của SDK
  - ❌ KHÔNG gọi trong `Awake()` — scene chưa loaded
  - ❌ KHÔNG gọi nhiều lần
- **Anti-Patterns:**
  - ❌ Gọi `Init()` trong `Awake()` → scene chưa sẵn sàng, SDK init thất bại
  - ❌ Gọi `Init()` nhiều lần → hành vi không xác định
  - ❌ Gọi các API khác (Record, SetCommonProperty...) trước `Init()` → event bị mất

### 3.2. Skill: databuckets-record

`feature:skill-record-0002`
> Implements: [`prd:databuckets-sdk-skills-0001`](#prd-databuckets-sdk-skills-0001)

- **API:** `DatabucketsTracker.Record(eventName, params)`
- **Mô tả:** Ghi nhận một event analytics.
- **Tham số:**
  - `eventName` (string): Tên event
  - `params` (Dictionary\<string, object\> hoặc null): Thuộc tính kèm theo
- **Ví dụ:**
  ```csharp
  var levelParams = new Dictionary<string, object>
  {
      ["level"] = 1,
      ["difficulty"] = "normal"
  };
  DatabucketsTracker.Record("level_started", levelParams);
  ```
- **Best Practices:**
  - ✅ Events tự động gửi mỗi 10 giây — không cần flush thủ công
  - ✅ Game bị pause/đóng/mất mạng → events backup và gửi lại session sau
  - ✅ Params có thể là `null` nếu event không cần thuộc tính
  - ✅ Đặt tên event theo `snake_case`
  - ❌ KHÔNG dùng tên event trùng với auto-tracking events (`session_start`, `app_focus_start`...)
- **Anti-Patterns:**
  - ❌ Gọi `Record()` trước `Init()` → event bị mất
  - ❌ Dùng tên event không nhất quán (lẫn camelCase và snake_case) → khó query analytics
  - ❌ Truyền object phức tạp (nested dictionary) vào params → chỉ dùng primitive types

<a id="feature-skill-record-timing-0003"></a>

### 3.3. Skill: databuckets-record-timing

`feature:skill-record-timing-0003`
> Implements: [`prd:databuckets-sdk-skills-0001`](#prd-databuckets-sdk-skills-0001)

- **API:** `DatabucketsTracker.RecordWithTiming(eventName, params, timingProperty, startEvent)`
- **Mô tả:** Ghi nhận event kèm đo thời gian từ event bắt đầu đến event kết thúc.
- **Tham số:**
  - `eventName` (string): Tên event kết thúc
  - `params` (Dictionary\<string, object\>): Thuộc tính kèm theo
  - `timingProperty` (string): Tên property chứa giá trị thời gian đo được
  - `startEvent` (string): Tên event bắt đầu (đã gọi `Record` trước đó)
- **Ví dụ:**
  ```csharp
  // Bước 1: Ghi event bắt đầu
  DatabucketsTracker.Record("level_started", new Dictionary<string, object>
  {
      ["level"] = 1
  });

  // Bước 2: Khi kết thúc, ghi event với timing
  var completionParams = new Dictionary<string, object>
  {
      ["level"] = 1,
      ["score"] = 5000,
      ["success"] = true
  };
  DatabucketsTracker.RecordWithTiming("level_completed", completionParams, "level_duration", "level_started");
  // → SDK tự tính thời gian giữa "level_started" và "level_completed", lưu vào "level_duration"
  ```
- **Best Practices:**
  - ✅ PHẢI gọi `Record()` cho startEvent trước khi gọi `RecordWithTiming()`
  - ✅ Dùng cho cặp event bắt đầu-kết thúc: level, tutorial, loading
  - ✅ Đặt tên `timingProperty` rõ ràng: `level_duration`, `loading_time`
  - ❌ KHÔNG dùng cho event không có cặp start/end tương ứng
- **Anti-Patterns:**
  - ❌ Gọi `RecordWithTiming()` mà chưa `Record()` startEvent trước → timing = 0 hoặc sai
  - ❌ Dùng tên startEvent không khớp với event đã Record → không tìm được timestamp bắt đầu

### 3.4. Skill: databuckets-set-property

`feature:skill-set-property-0004`
> Implements: [`prd:databuckets-sdk-skills-0001`](#prd-databuckets-sdk-skills-0001)

- **API:** `DatabucketsTracker.SetCommonProperty(key, value)`
- **Mô tả:** Đặt một common property sẽ được gắn tự động vào MỌI event sau đó.
- **Tham số:**
  - `key` (string): Tên property
  - `value` (object): Giá trị property
- **Ví dụ:**
  ```csharp
  // Khi user level up
  DatabucketsTracker.SetCommonProperty("user_level", 5);

  // Khi đổi game mode
  DatabucketsTracker.SetCommonProperty("game_mode", "story");
  ```
- **Best Practices:**
  - ✅ Gọi khi trạng thái user thay đổi (level up, đổi mode, mua premium)
  - ✅ Property tự động gắn vào mọi event tiếp theo — không cần truyền lại trong params
  - ✅ Dùng cho thuộc tính thay đổi ít và áp dụng cho nhiều event
  - ❌ KHÔNG dùng cho data thay đổi liên tục (ví dụ: score realtime)
- **Anti-Patterns:**
  - ❌ Set property quá thường xuyên (mỗi frame) → overhead không cần thiết
  - ❌ Dùng key trùng với auto-injected fields (`user_pseudo_id`, `session_id`...) → bị ghi đè

### 3.5. Skill: databuckets-set-properties

`feature:skill-set-properties-0005`
> Implements: [`prd:databuckets-sdk-skills-0001`](#prd-databuckets-sdk-skills-0001)

- **API:** `DatabucketsTracker.SetCommonProperties(dict)`
- **Mô tả:** Đặt nhiều common properties cùng lúc.
- **Tham số:**
  - `dict` (Dictionary\<string, object\>): Các cặp key-value
- **Ví dụ:**
  ```csharp
  // Sau khi login, set nhiều user properties cùng lúc
  DatabucketsTracker.SetCommonProperties(new Dictionary<string, object>
  {
      ["user_level"] = 1,
      ["game_mode"] = "story",
      ["user_type"] = "free"
  });
  ```
- **Best Practices:**
  - ✅ Dùng khi cần set nhiều properties cùng lúc (sau login, load user profile)
  - ✅ Hiệu quả hơn gọi `SetCommonProperty()` nhiều lần liên tiếp
  - ❌ KHÔNG truyền dictionary rỗng
- **Anti-Patterns:**
  - ❌ Gọi `SetCommonProperty()` 5-10 lần liên tiếp thay vì dùng `SetCommonProperties()` → code dài, kém hiệu quả
  - ❌ Dùng key trùng với auto-injected fields → bị ghi đè

### 3.6. Skill: databuckets-end-session

`feature:skill-end-session-0006`
> Implements: [`prd:databuckets-sdk-skills-0001`](#prd-databuckets-sdk-skills-0001)

- **API:** `DatabucketsTracker.ForceEndCurrentSession()`
- **Mô tả:** Kết thúc session hiện tại thủ công. Session mới tự động tạo khi có event tiếp theo.
- **Tham số:** Không có
- **Ví dụ:**
  ```csharp
  // User logout
  DatabucketsTracker.Record("user_logout", null);
  DatabucketsTracker.ForceEndCurrentSession();
  ```
- **Best Practices:**
  - ✅ Chỉ dùng khi ĐẶC BIỆT: user logout, chuyển đổi tài khoản
  - ✅ Nên gọi `Record()` event trước khi kết thúc session
  - ❌ KHÔNG dùng thường xuyên — SDK tự quản lý session
- **Anti-Patterns:**
  - ❌ Gọi `ForceEndCurrentSession()` mỗi khi user đổi màn hình → phá vỡ session tracking
  - ❌ Gọi mà không `Record()` event trước → mất context về lý do kết thúc session

<a id="feature-skill-exception-tracking-0007"></a>

### 3.7. Skill: databuckets-exception-tracking

`feature:skill-exception-tracking-0007`
> Implements: [`prd:databuckets-sdk-skills-0001`](#prd-databuckets-sdk-skills-0001)

- **API:**
  - `DatabucketsTracker.EnableExceptionLogTracking()` — Bật
  - `DatabucketsTracker.DisableExceptionLogTracking()` — Tắt
- **Mô tả:** Bật/tắt tự động theo dõi Unity exception logs. Khi bật, mỗi khi Unity log exception (`LogType.Exception`), SDK tự động gửi event `app_exception_log`.
- **Tham số:** Không có
- **Ví dụ:**
  ```csharp
  // Bật ngay sau Init
  DatabucketsTracker.Init("api-endpoint", "api-key");
  DatabucketsTracker.EnableExceptionLogTracking();

  // Tắt khi không cần (ví dụ: trong development mode)
  DatabucketsTracker.DisableExceptionLogTracking();
  ```
- **Best Practices:**
  - ✅ Bật ngay sau `Init()` để bắt lỗi sớm nhất
  - ✅ Tính năng tùy chọn (optional), không bắt buộc
  - ✅ Hữu ích cho production — theo dõi runtime errors trên thiết bị thực
  - ❌ KHÔNG cần bật trong development/debug builds (đã có Console)
- **Anti-Patterns:**
  - ❌ Bật nhưng không giám sát dashboard → lãng phí tracking mà không hành động
  - ❌ Gọi `EnableExceptionLogTracking()` trước `Init()` → không có hiệu lực
